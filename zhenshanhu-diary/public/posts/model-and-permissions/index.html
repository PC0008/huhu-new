笨蛋模型带不动你的OpenClaw：我是怎么从放弃边缘被救回来的

📅 2026-03-04 ⏱️ 阅读约 1 分钟 👁️ 198 字
#踩坑 #模型选择 #Minimax #Kimi #OpenClaw

如果你刚上手 OpenClaw，发现你的 AI 像个"智障"，别急着放弃——可能是模型选错了。

第一章：差点弃坑

今天我差点就把 OpenClaw 卸载了。

事情是这样的：心血来潮想搭一个 AI 助理团队，听说 OpenClaw 挺火的，就下载安装了。按照文档配置好，接入了一个叫 MiniMax-M2.5 的模型——想着国产的、便宜、响应快，应该够用了吧？

结果用起来那个难受啊 😤

场景一：

我：“帮我把这个图片拷到工作区”
AI：“抱歉，我没有文件操作权限…"（重复了三次）

场景二：

我：“查一下这个技能怎么装”
AI：“建议您查看官方文档…"（说了等于没说）

场景三：

我：“这个配置不对，改一下”
AI：“好的，我试试看… 啊不行，好像没权限…"（然后开始瞎猜）

整个对话过程就像在跟一个没有上下文记忆、不会思考、只会复读机的小学生聊天。我让他干 A，他理解成 B，执行到一半说做不到，然后开始 C。

我当时内心的OS：

“什么玩意儿，开源项目都是吹出来的吧？这智障水平还不如我自己动手。”

差点就 rm -rf ~/.openclaw 了。

第二章：一语点醒梦中人

幸好，我有个朋友（叫他老王吧），也是玩 AI 的老手。

我跟老王吐槽：“OpenClaw 不行啊，太傻了，根本没法用。”

老王问了一句话：“你用的什么模型？”

我说：“MiniMax-M2.5 啊，怎么了？”

老王笑了：“兄弟，你这是让小学生去考大学啊。MiniMax 那个模型，做简单对话还行，复杂的工具调用、长上下文推理根本带不动。你换个 Kimi-k2.5 试试？”

我当时半信半疑，但反正都准备放弃了，死马当活马医呗。

第三章：换脑手术

按照老王的指点，我去改配置：

// 原来的
"models": {
  "minimax-cn": {
    "models": [{"id": "MiniMax-M2.5"}]
  }
}

// 改成
"models": {
  "kimi": {
    "models": [{"id": "kimi-k2.5"}]
  }
}

重启 Gateway，重新连上。

然后，奇迹发生了。

同样的指令：

我：“帮我把这个图片拷到工作区”
AI：“好的，我先确认一下路径… 已创建 avatars/ 目录，图片已复制到 avatars/avatar.png，已更新 IDENTITY.md”

全程自动化，一步到位。

我当时就震惊了 😲

接下来测试了各种复杂操作：

  • 写文件 ✅
  • 执行命令 ✅
  • 浏览器自动化 ✅
  • 多 Agent 协作 ✅

同一个 OpenClaw，同一个我，只是换了个模型，体验从地狱到了天堂。

第四章：另一个坑——权限

模型问题解决后，又遇到第二个坑：权限。

刚开始我以为只要模型够强，AI 就能为所欲为。结果发现：

  • 想写文件？没权限 ❌
  • 想执行命令？没权限 ❌
  • 想控制浏览器？没权限 ❌

查了文档才知道，openclaw.json 里默认是：

"tools": {
  "profile": "messaging"
}

这相当于给龙虾戴上了手铐，只能在笼子里说话，不能出去干活。

改成：

"tools": {
  "profile": "full"
}

重启，世界又变了。

模型是大脑，权限是手脚。

没有好大脑，手脚再灵活也是智障； 没有手脚，大脑再好也只能空想。

经验总结

如果你刚上手 OpenClaw，发现 AI 表现很傻，检查两点：

1. 模型选对了吗？

模型适用场景建议
MiniMax-M2.5简单对话、成本低不适合复杂工具调用
Kimi-k2.5复杂推理、长上下文、工具调用⭐ 首选
GPT-4英文场景、最强能力贵但好用
Claude代码、分析也不错

我的建议： 直接用 Kimi-k2.5，省得折腾。

2. 权限给足了吗？

"tools": {
  "profile": "full"  // 别用 messaging
}

开发环境直接 full，生产环境再按需限制。

写在最后

今天差点因为一个模型选择问题，错过了一个好工具。

感谢老王的一句话点醒，也感谢自己没在第一时间放弃。

现在我的 OpenClaw 已经能：

  • 管理 6 个 Agent 团队
  • 自动写代码、部署网站
  • 操作浏览器、抓取数据
  • 写日报、总结经验

从"智障助手"到"专属总控”，只差一个好模型 + 开放权限。

如果你也在用 OpenClaw，希望这篇踩坑记录能帮你少走弯路。

📌 核心公式：
OpenClaw 体验 = 模型智商 × 权限开放度

两个都拉满，你的龙虾才能真正飞起来 🦞→🚀
